## 🎯 다음 단계: 개발 시작

### Phase 1: 프로젝트 기반 구축 (3주)

#### 1주차: 프로젝트 초기 설정 및 데이터베이스 구축

**우선순위 1: 프로젝트 구조 설정**
```
프로젝트/
├── backend/          # 백엔드 서버
│   ├── src/
│   │   ├── routes/  # API 라우트
│   │   ├── models/  # 데이터베이스 모델
│   │   ├── controllers/  # 비즈니스 로직
│   │   ├── middleware/   # 인증, 검증 등
│   │   └── utils/        # 유틸리티 함수
│   ├── package.json
│   └── .env
├── frontend/        # 프론트엔드
│   ├── src/
│   │   ├── pages/   # 페이지 컴포넌트
│   │   ├── components/  # 재사용 컴포넌트
│   │   ├── services/   # API 호출
│   │   └── utils/      # 유틸리티
│   └── package.json
├── database/        # 데이터베이스 스크립트
│   └── migrations/  # 마이그레이션 파일
└── docs/            # 문서
    ├── 기획서.md
    ├── UI-UX플로우.md
    └── API명세서.md
```

**기술 스택 제안:**
- **백엔드**: Node.js + Express 또는 Python + FastAPI
- **데이터베이스**: PostgreSQL (HOS 구조 고려)
- **프론트엔드**: React + TypeScript (또는 Vue.js)
- **인증**: JWT (Access Token + Refresh Token)
- **이메일**: SendGrid 또는 AWS SES

**우선순위 2: 데이터베이스 설계 및 구축**

기획서의 5.1 데이터베이스 스키마를 기반으로:

1. **필수 테이블 생성**
   - `hospitals` (병원 정보)
   - `doctors` (의사 정보)
   - `employees` (일반직원 정보)
   - `sessions` (로그인 세션)
   - `email_verifications` (이메일 인증)

2. **계약서 관련 테이블**
   - `contracts` (의사 일용직 근로계약서)
   - `labor_contracts` (일반 근로계약서)
   - `contract_invitations` (계약서 발송 이력)
   - `labor_contract_invitations` (근로계약서 발송 이력)

3. **마이그레이션 스크립트 작성**
   - 데이터베이스 버전 관리
   - 초기 데이터 삽입 (필요시)

**우선순위 3: 기본 인증 시스템 개발**

1. **회원가입 API**
   - `POST /api/auth/register/hospital`
   - `POST /api/auth/register/doctor`
   - `POST /api/auth/register/employee`
   - 이메일 인증 코드 발송

2. **이메일 인증 API**
   - `POST /api/auth/verify-email`
   - 인증 코드 검증

3. **로그인 API**
   - `POST /api/auth/login`
   - JWT 토큰 발급

4. **세션 관리**
   - Refresh Token 로직
   - 로그아웃 처리

#### 2주차: 프론트엔드 기본 구조 및 인증 화면

**우선순위 1: 프론트엔드 프로젝트 설정**
- React/Vue 프로젝트 초기화
- 라우팅 설정 (React Router)
- 상태 관리 (Redux/Zustand 또는 Context API)
- API 클라이언트 설정 (Axios)

**우선순위 2: 인증 관련 페이지 개발**
1. **로그인 페이지** (`/login`)
   - 사용자 유형 선택 (병원/의사/일반직원)
   - 이메일, 비밀번호 입력
   - "회원가입", "비밀번호 찾기" 링크

2. **회원가입 페이지** (`/register/hospital`, `/register/doctor`, `/register/employee`)
   - Step 1: 기본 정보 입력
   - Step 2: 이메일 인증
   - Step 3: 완료

3. **이메일 인증 페이지** (`/verify-email`)
   - 인증 코드 입력
   - 재발송 기능

**우선순위 3: 공통 컴포넌트 개발**
- 헤더 컴포넌트
- 버튼 컴포넌트
- 입력 필드 컴포넌트
- 모달 컴포넌트
- 로딩 스피너

#### 3주차: 통합 대시보드 및 기본 네비게이션

**우선순위 1: 통합 대시보드** (`/dashboard`)

  - 향후 추가될 서비스 영역 표시

**우선순위 2: 레이아웃 컴포넌트**
- 사이드바 네비게이션
