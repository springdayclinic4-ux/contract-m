# 의사 일용직 근로계약서 생성 및 전자서명 시스템 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**근로계약서 생성 및 전자서명 통합 시스템** (HOS Domain 내 계약서 관리 모듈)

### 1.2 프로젝트 목적
병원, 의사, 일반직원이 각각 계정을 가지고, 병원이 다양한 근로계약서를 생성하여 발송하고 근로자가 전자서명할 수 있는 통합 웹 애플리케이션. 이 시스템은 향후 확장될 HOS (Hospital Operating System) 플랫폼의 일부로 설계되며, 조직 관리, 권한 관리, 멤버 관리 등과 통합될 예정입니다.

### 1.3 주요 목표
- 병원, 의사, 일반직원 계정 분리 회원가입 시스템 구축
- 병원 정보 자동 연동으로 반복 입력 최소화
- 이메일 기반 계약서 발송 시스템
- 전자서명 기능 제공
- 계약서 생성, 발송, 서명, 관리 통합 플로우
- 계약서 통계 및 분석 기능 제공
- 다양한 근로계약서 유형 지원 (의사 일용직, 일반 근로계약서)
- **HOS 플랫폼 아키텍처 기반 확장 가능한 구조 설계**

### 1.4 플랫폼 아키텍처 개요

**참고**: 상세한 HOS Data Structure는 `HOS_Data_Structure.md` 파일을 참고하세요.

본 시스템은 향후 확장될 **Platform** 구조의 일부로 설계됩니다. **HOS 구조가 메인 아키텍처**이며, 모든 데이터베이스 설계와 기능은 이 구조를 기반으로 합니다:

```
Platform
├── 사용자 Domain (User Domain)
│   └── 개인정보 본인인증, 계정 인증 등
│
├── HOS Domain (Hospital Operating System) ← 현재 시스템
│   ├── 조직 DB
│   │   ├── Organization 데이터
│   │   │   ├── Member Information (멤버 소속 정보)
│   │   │   ├── Organization Definition (조직 정의)
│   │   │   ├── Organization Operating Information (조직 운영 정보)
│   │   │   ├── Role (역할 정의 및 권한 프리셋)
│   │   │   ├── Team (팀 정의)
│   │   │   └── Member_Team_Role Assignment (멤버-팀-역할 배정)
│   │   └── 권한 관리 시스템
│   │       ├── Permission (시스템 권한 정의)
│   │       ├── Member Permission Set (개별 권한 추가/제외)
│   │       └── Member_Permission History (권한 변경 이력)
│   │
│   ├── 계약서 관리 모듈 (현재 개발 중) ← 본 시스템
│   │   ├── 의사 일용직 근로계약서
│   │   └── 일반 근로계약서
│   │
│   ├── Customer 데이터 (향후 확장)
│   ├── Operation 데이터 (예약, 방문 관리 등, 향후 확장)
│   ├── Consultation 데이터 (상담 관리, 향후 확장)
│   ├── Product 데이터 (상품 관리, 향후 확장)
│   ├── Procedure 데이터 (시술 관리, 향후 확장)
│   ├── Finance 데이터 (결제, 환불 관리, 향후 확장)
│   └── 인사 데이터 (인건비, 성과, 연차 등, 향후 확장)
│
├── 메신저 Domain (Messenger Domain, 향후 확장)
└── B2C Domain (향후 확장)
```

**시스템 흐름:**
1. 사용자 회원가입 (개인정보 본인인증) → 사용자 Domain
2. 사용자 로그인 (계정 인증) → 사용자 Domain
3. HOS 접속 후 시스템 사용 → HOS Domain
   - 소속 조직 확인
   - Session 발급
   - 조직별 권한 확인
   - 계약서 관리 모듈 접근

**확장성 고려사항:**
- 한 병원에서 여러 진료 과목 운영 가능 (피부 미용, 정형외과 등)
- 조직 계층 구조 지원 (MSO > 병원)
- 멤버별 개별 권한 추가/제외 가능
- 팀 및 역할 기반 권한 관리
- 향후 다양한 HOS 모듈 추가 가능

## 2. 사용자 유형

### 2.1 병원 (갑)
- 병원 계정으로 회원가입
- 의사 일용직 근로계약서 생성 및 의사에게 발송
- 일반 근로계약서 생성 및 일반직원에게 발송
- 계약서 관리 및 통계 확인

### 2.2 의사 (갑/을)
- 의사 계정으로 회원가입
- 병원/일반직원/다른 의사로부터 받은 일용직 근로계약서 확인
- 전자서명 및 승인/거부
- **의사 일용직 근로계약서 생성 및 다른 의사/일반직원에게 발송 가능**
- **일반 근로계약서 생성 및 일반직원에게 발송 가능**

### 2.3 일반직원 (갑/을)
- 일반직원 계정으로 회원가입
- 병원/의사/다른 일반직원으로부터 받은 근로계약서 확인
- 전자서명 및 승인/거부
- **일반 근로계약서 생성 및 다른 일반직원/의사에게 발송 가능**
- **의사 일용직 근로계약서 생성 및 의사에게 발송 가능**

## 3. 핵심 기능

### 3.1 병원 회원가입 기능

#### 3.1.1 회원가입 화면
- **입력 항목**
  - 병원명 (필수)
  - 사업자등록번호 (필수, 중복 체크, ID로 사용)
  - 대표자 성명 (필수)
  - 병원 주소 (필수)
  - 병원 연락처 (필수)
  - 담당자명 (선택)
  - 담당자 연락처 (선택, 담당자명 입력 시 필수)
  - 이메일 (필수, 중복 체크, 이메일 인증)
  - 비밀번호 (필수, 8자 이상, 영문+숫자+특수문자 조합)
  - 비밀번호 확인 (필수)
  - 약관 동의 (필수)
    - 서비스 이용약관 (병원용) (필수)
    - 개인정보 처리방침 (병원용) (필수)
    - 개인정보 제3자 제공 동의 (필수)
    - 마케팅 정보 수신 동의 (선택)

#### 3.1.2 회원가입 프로세스
1. 기본 정보 입력
2. 이메일 인증 (인증 코드 발송)
3. 이메일 인증 완료
4. 회원가입 완료 및 자동 로그인

### 3.2 의사 회원가입 기능

#### 3.2.1 회원가입 화면
- **입력 항목**
  - 성명 (필수)
  - 이메일 (필수, 중복 체크, 인증 필요)
  - 면허번호 (필수)
  - 주소 (필수)
  - 연락처 (선택, 있으면 입력)
  - 비밀번호 (필수, 8자 이상, 영문+숫자+특수문자 조합)
  - 비밀번호 확인 (필수)
  - 은행명 (선택)
  - 계좌번호 (선택)
  - 약관 동의 (필수)
    - 서비스 이용약관 (의사용) (필수)
    - 개인정보 처리방침 (의사용) (필수)
    - 마케팅 정보 수신 동의 (선택)
  
**참고**: 
- 주민등록번호는 회원가입 시 수집하지 않으며, 계약서 작성 시 작성자가 입력합니다.
- 개인정보 제공 동의는 계약서 포맷에 이미 포함되어 있으므로 별도로 받지 않습니다.

#### 3.2.2 회원가입 프로세스
1. 기본 정보 입력
2. 이메일 인증 (인증 코드 발송)
3. 이메일 인증 완료
4. 회원가입 완료 및 자동 로그인

**특별 플로우**: 병원이 계약서를 이메일로 발송하면, 의사가 이메일 링크를 클릭하여 계약서를 확인하고, 미가입 의사인 경우 회원가입 페이지로 이동하여 바로 회원가입할 수 있습니다.

  - 약관 동의 (필수)

### 3.4 의사 일용직 근로계약서 생성 및 발송 기능

#### 3.4.1 계약서 생성
- 병원/의사/일반직원이 로그인 후 계약서 생성 페이지 접속
- 기존 index.html 기능 유지
- 작성자 정보 자동 로드 (병원 정보 또는 작성자 정보)
- 의사 정보 입력:
  - **의사 이메일 입력 (핵심)**
  - 이메일로 의사 계정 검색
  - 의사가 가입되어 있으면 기본 정보 자동 불러오기 (성명, 면허번호, 주소, 연락처, 계좌정보)
  - 의사가 가입되어 있지 않으면 수동 입력
  - **주민등록번호 입력** (계약서 작성 시 필수)
- 근무 조건, 급여 정보 입력
- 계약서 미리보기 확인

#### 3.4.2 계약서 발송
- "의사에게 발송" 버튼 클릭
- **이메일 발송** (의사 이메일로 계약서 링크 발송)
- 발송 완료 후 계약서 상태: "발송됨" → "서명 대기"
- 의사가 이메일 링크 클릭 시:
  - 의사가 가입되어 있으면: 계약서 확인 페이지로 이동
  - 의사가 가입되어 있지 않으면: 회원가입 페이지로 이동 (이메일 자동 입력, 계약서 확인 후 회원가입 유도)

### 3.4 전자서명 기능

#### 3.6.1 의사 계약서 확인
- 의사가 로그인 후 "받은 계약서" 메뉴 접속
- 또는 이메일 링크 클릭 시 계약서 확인 페이지 접속
- **의사가 미가입인 경우**:
  - 이메일 링크 클릭 시 회원가입 페이지로 이동
  - 이메일 자동 입력 (계약서에 포함된 이메일)
  - 회원가입 완료 후 자동으로 계약서 확인 페이지로 이동
- 계약서 내용 전체 확인
- PDF 미리보기

#### 3.6.2 의사 전자서명 프로세스
1. 의사가 계약서 내용 확인
   - 계약서에 포함된 개인정보 제공 동의서 확인 (계약서 포맷 내 포함)
2. "서명하기" 버튼 클릭
3. 서명 방법 선택:
   - 터치/마우스로 직접 서명
   - 서명 이미지 업로드
   - 도장 이미지 업로드
4. 서명 완료 후 "승인" 또는 "거부" 선택
5. 승인 시:
   - 계약서 상태: "서명 완료"
   - 병원에게 알림 발송
   - 계약서 PDF 자동 생성 및 저장
6. 거부 시:
   - 거부 사유 입력 (선택)
   - 계약서 상태: "거부됨"
   - 병원에게 알림 발송

### 3.7 계약서 관리 기능

#### 3.7.1 병원/의사/일반직원 - 계약서 관리 (공통)
- **작성한 계약서 관리**
  - 계약서 목록 (초안, 발송 대기, 서명 대기, 서명 완료, 거부됨)
  - 계약서 상태별 필터링
  - 수신자별, 날짜별 검색
  - 계약서 상세 보기
  - 서명 완료된 계약서 PDF 다운로드
  - 계약서 수정 (서명 전까지 가능)
  - 계약서 재발송

- **받은 계약서 관리**
  - 받은 계약서 목록 (서명 대기, 승인, 거부)
  - 계약서 상세 보기
  - 서명 완료된 계약서 PDF 다운로드
  - 계약서 이력 확인

## 5. 데이터베이스 설계

### 5.1 테이블 구조

#### 5.1.1 hospitals (병원 계정) - 현재 구현

**참고**: 향후 HOS 플랫폼 확장 시 이 테이블은 `organizations` 테이블과 통합되며, `member_information` 테이블을 통해 사용자와 연결됩니다.

### 5.2 향후 HOS 플랫폼 통합 시 데이터 구조

#### 5.2.1 계약서 모듈과 조직 DB의 연계

향후 HOS 플랫폼 확장 시 다음과 같이 통합됩니다:

1. **조직 중심 계약서 관리**
   - `contracts`, `labor_contracts` 테이블의 `creator_id`는 `member_information.id`와 연결
   - 조직별 계약서 통계 및 관리 가능
   - 멤버의 계약 이력이 `member_information`에 연결되어 인사 데이터로 활용

2. **권한 기반 계약서 접근**
   - `Permission` 테이블의 권한 키 예시:
     - `HOS:Contract:CREATE` - 계약서 생성 권한
     - `HOS:Contract:VIEW` - 계약서 조회 권한
     - `HOS:Contract:EDIT` - 계약서 수정 권한
     - `HOS:Contract:DELETE` - 계약서 삭제 권한
     - `HOS:Contract:SEND` - 계약서 발송 권한
     - `HOS:Contract:SIGN` - 계약서 서명 권한
   - `Role`의 권한 프리셋 또는 `Member Permission Set`을 통해 개별 권한 부여/제거

3. **팀 및 역할 기반 계약서 관리**
   - 특정 팀(예: 인사팀)만 계약서 생성 가능하도록 설정
   - 역할(예: 인사담당자)별로 다른 계약서 접근 권한 부여

4. **멤버 소속 정보 활용**
   - 멤버의 입사일, 퇴사일 정보를 계약서와 연계
   - 멤버의 급여 정보와 계약서의 급여 정보 연동
   - 멤버의 성과 정보와 계약서 이력 연계

#### 5.2.2 확장 가능한 모듈 구조

현재 계약서 관리 모듈은 향후 다음과 같은 HOS 모듈들과 통합될 수 있습니다:

- **인사 데이터 모듈**: 계약서 → 급여 정보, 성과 정보, 연차 정보와 연계
- **Customer 데이터 모듈**: (향후 확장)
- **Operation 데이터 모듈**: 예약, 방문 관리와 연계 (향후 확장)
- **Consultation 데이터 모듈**: 상담 관리와 연계 (향후 확장)
- **Product 데이터 모듈**: 상품 관리와 연계 (향후 확장)
- **Procedure 데이터 모듈**: 시술 관리와 연계 (향후 확장)
- **Finance 데이터 모듈**: 결제, 환불 관리와 연계 (향후 확장)

#### 5.2.3 데이터 마이그레이션 계획

향후 HOS 플랫폼 통합 시 데이터 마이그레이션 계획:

1. **병원 데이터 마이그레이션**
   - `hospitals` 테이블 → `organizations` 테이블
   - 병원 정보는 `organization_definition`에 저장
   - 병원 운영 정보는 `organization_operating_information`에 저장

2. **의사/일반직원 데이터 마이그레이션**
   - `doctors`, `employees` 테이블 → 사용자 Domain의 `personal_information` 테이블
   - 조직 소속 정보는 `member_information` 테이블에 생성
   - 멤버 상태는 '근무', '퇴사', '휴직'으로 관리

3. **계약서 데이터 연계**
   - `contracts`, `labor_contracts` 테이블의 `creator_id`를 `member_information.id`로 연결
   - 계약서는 멤버의 인사 이력으로 관리
   - 조직별 계약서 통계 및 관리 기능 추가

4. **권한 시스템 통합**
   - 기존 사용자별 접근 제어 → Role, Team, Permission 기반 권한 관리
   - 멤버별 개별 권한 추가/제외 기능 제공

### 10.3 일용소득세율 관련 사항 (2025~2026년 귀속 기준)

- **일용소득세 계산 방식**
  1. **근로소득공제**: 일 150,000원
  2. **소득세율**: 6%
  3. **근로소득 세액공제**: 산출세액의 55%
  4. **실질 세율**: 약 2.7% (공제 후 초과금액 기준)
  5. **소액부징수**: 산출된 소득세가 1,000원 미만인 경우 징수하지 않음 (일당 187,000원까지는 세금 없음)

- **계산 공식**
  - 과세표준 = 일당 - 150,000원
  - 산출세액 = 과세표준 × 6%
  - 소득세 = 산출세액 × 45% (10원 단위 절사)
  - 지방소득세 = 소득세 × 10% (10원 단위 절사)
  - 실수령액 = 일당 - (소득세 + 지방소득세)

- **적용 기준**
  - 일용근로자(동일 고용주에게 3개월 미만 근무)에게 적용
  - 일급 지급 시 원천징수 방식으로 세금 공제
  
- **주의사항**
  - 동일 고용주에게 3개월 이상 계속 근무 시 상용근로자로 분류되어 일반 근로소득세율(6%~45% 누진세) 적용 가능
  - 월 60시간 이상 또는 월 8일 이상 근무 시 4대보험 가입 의무 발생 가능
  - 계산은 국고금단수계산법에 따라 10원 단위로 절사 처리
