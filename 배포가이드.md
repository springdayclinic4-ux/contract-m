## 방법 1: Vercel + Render (추천) ⭐

### 장점
- 무료 플랜으로 충분
- 가장 안정적
- 프론트엔드와 백엔드 분리로 관리 쉬움

### 1단계: 데이터베이스 준비 (Neon PostgreSQL)

#### 1-1. Neon 가입
1. https://neon.tech 접속
2. "Sign Up" 클릭 → GitHub으로 로그인
3. 새 프로젝트 생성

#### 1-2. 데이터베이스 생성
1. 프로젝트 이름: `theranova-db`
2. Region: `AWS / Asia Pacific (Seoul)` 선택
3. "Create Project" 클릭

#### 1-3. 연결 정보 복사
화면에 나타나는 `Connection String`을 복사하세요.
```
postgresql://username:password@ep-xxx-xxx.ap-southeast-1.aws.neon.tech/dbname?sslmode=require
```

---

### 2단계: 백엔드 배포 (Render)

#### 2-1. Render 가입
1. https://render.com 접속
2. "Get Started" → GitHub으로 로그인

#### 2-2. 백엔드 서비스 생성
1. Dashboard에서 "New +" 클릭
2. "Web Service" 선택
3. GitHub 저장소 연결
4. 다음 정보 입력:
   - **Name**: `theranova-backend`
   - **Region**: `Singapore (Southeast Asia)`
   - **Root Directory**: `backend`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`

#### 2-3. 환경 변수 설정
"Environment" 탭에서 다음 변수들을 추가:

```
NODE_ENV=production
PORT=3001

# Neon에서 복사한 DATABASE_URL
DATABASE_URL=postgresql://username:password@...

# JWT 비밀키 (랜덤 문자열 - 최소 32자)
JWT_SECRET=your-super-secret-jwt-key-change-this-to-random-string-32chars
JWT_REFRESH_SECRET=your-super-secret-refresh-key-also-change-this-32chars

# CORS 설정 (나중에 Vercel URL로 변경)
ALLOWED_ORIGINS=*

# 이메일 설정 (개발용 - 일단 목업)
MOCK_EMAIL=true
```

#### 2-4. 배포 시작
1. "Create Web Service" 클릭
2. 배포 완료까지 3-5분 대기
3. 배포 완료 후 URL 복사 (예: `https://theranova-backend.onrender.com`)

---

### 3단계: 데이터베이스 마이그레이션

#### 3-1. Render 대시보드에서
1. 배포된 서비스 클릭
2. "Shell" 탭 클릭
3. 다음 명령어 실행:

```bash
# Prisma 클라이언트 생성
npx prisma generate

# 데이터베이스 마이그레이션
npx prisma migrate deploy

# 데이터베이스 초기화 (선택사항)
npx prisma db push
```

---

### 4단계: 프론트엔드 배포 (Vercel)

#### 4-1. Vercel 가입
1. https://vercel.com 접속
2. "Sign Up" → GitHub으로 로그인

#### 4-2. 프로젝트 Import
1. "Add New..." → "Project" 클릭
2. GitHub 저장소 선택
3. "Import" 클릭

#### 4-3. 프로젝트 설정
1. **Framework Preset**: `Vite`
2. **Root Directory**: `frontend` (Edit 클릭해서 변경)
3. **Build Command**: `npm run build`
4. **Output Directory**: `dist`

#### 4-4. 환경 변수 설정
"Environment Variables" 섹션에서 추가:

```
VITE_API_URL=https://theranova-backend.onrender.com/api
```
(위에서 복사한 Render 백엔드 URL 사용)

#### 4-5. 배포
1. "Deploy" 클릭
2. 배포 완료까지 2-3분 대기
3. 배포 완료 후 URL 확인 (예: `https://your-app.vercel.app`)

---

### 5단계: CORS 설정 업데이트

#### 5-1. Render로 돌아가기
1. Render 대시보드 → 백엔드 서비스 선택
2. "Environment" 탭
3. `ALLOWED_ORIGINS` 변수를 Vercel URL로 변경:

```
ALLOWED_ORIGINS=https://your-app.vercel.app
```

4. "Save Changes" 클릭 → 자동 재배포

---

### 6단계: 완료! 🎉

이제 `https://your-app.vercel.app`에 접속하면 작동합니다!

## 방법 2: Railway (올인원) 🚂

### 장점
- 한 곳에서 모두 관리
- 설정이 간단
- PostgreSQL 자동 연결

### 단점
- 무료 플랜 제한 ($5 크레딧)

### 1단계: Railway 가입
1. https://railway.app 접속
2. "Login" → GitHub으로 로그인

### 2단계: 새 프로젝트 생성
1. "New Project" 클릭
2. "Deploy from GitHub repo" 선택
3. 저장소 선택

### 3단계: PostgreSQL 추가
1. "New" 클릭
2. "Database" → "PostgreSQL" 선택

### 4단계: 백엔드 서비스 설정
1. GitHub repo 클릭
2. "Settings" 탭
   - **Root Directory**: `backend`
   - **Build Command**: `npm install && npx prisma generate`
   - **Start Command**: `npm start`

### 5단계: 환경 변수 설정
"Variables" 탭에서 자동으로 `DATABASE_URL`이 설정됩니다.
추가로 다음을 입력:

```
NODE_ENV=production
PORT=3001
JWT_SECRET=your-super-secret-jwt-key-32chars
JWT_REFRESH_SECRET=your-super-secret-refresh-key-32chars
MOCK_EMAIL=true
ALLOWED_ORIGINS=*
```

### 6단계: 프론트엔드 서비스 추가
1. "New" 클릭
2. "GitHub Repo" 선택 (같은 저장소)
3. "Settings" 탭
   - **Root Directory**: `frontend`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npx serve -s dist -l $PORT`

### 7단계: 프론트엔드 환경 변수
"Variables" 탭:
```
VITE_API_URL=https://your-backend.railway.app/api
```
(백엔드 서비스의 URL 사용)

### 8단계: 데이터베이스 마이그레이션
백엔드 서비스에서 "Deploy logs" 확인 후:
```bash
# Railway CLI 설치
npm install -g @railway/cli

# 로그인
railway login

# 프로젝트 선택
railway link

# 마이그레이션
railway run npx prisma migrate deploy
```

## 방법 3: 간단 테스트용 (로컬) 💻

서버에 올리기 전에 로컬에서 먼저 테스트하기:

### 1단계: PostgreSQL 설치
1. https://www.postgresql.org/download/ 접속
2. Windows용 설치 프로그램 다운로드
3. 설치 시 비밀번호 설정 (꼭 기억하세요!)

### 2단계: 데이터베이스 생성
```sql
-- pgAdmin 또는 psql에서 실행
CREATE DATABASE theranova;
```

### 3단계: 환경 변수 설정
`backend/.env` 파일 생성:
```
DATABASE_URL="postgresql://postgres:your-password@localhost:5432/theranova"
JWT_SECRET="your-secret-key-at-least-32-characters-long"
JWT_REFRESH_SECRET="your-refresh-secret-also-32-chars"
ALLOW_UNVERIFIED_SIGNUP=true
MOCK_EMAIL=true
```

### 4단계: 실행
```bash
# 터미널 1 - 백엔드
cd backend
npm install
npm run db:generate
npm run db:migrate
npm run dev

# 터미널 2 - 프론트엔드
cd frontend
npm install
npm run dev
```

### 5단계: 접속
http://localhost:5173 에 접속

## 🆘 문제 해결

### 백엔드 연결 안됨
1. Render/Railway 로그 확인
2. `DATABASE_URL` 환경변수 확인
3. CORS 설정 확인

### 프론트엔드 빌드 실패
1. `VITE_API_URL` 환경변수 확인
2. Node.js 버전 확인 (18 이상 필요)

### 데이터베이스 마이그레이션 실패
```bash
# Prisma 초기화
npx prisma migrate reset --force
npx prisma migrate deploy
```
