# 데이터베이스 인증 오류 해결 가이드

## 🔍 오류 원인

```
Error: P1000: Authentication failed against database server
```

이 오류는 다음 중 하나일 수 있습니다:

1. **PostgreSQL 비밀번호가 다름**
   - `.env` 파일의 `DATABASE_URL`에서 `password123`이 실제 PostgreSQL 비밀번호와 다를 수 있습니다

2. **데이터베이스가 존재하지 않음**
   - `hos_contracts` 데이터베이스가 아직 생성되지 않았을 수 있습니다

3. **PostgreSQL 서버가 실행되지 않음**
   - PostgreSQL 서비스가 시작되지 않았을 수 있습니다

## ✅ 해결 방법

### 1. PostgreSQL 비밀번호 확인

`.env` 파일의 `DATABASE_URL`을 확인하고, 실제 PostgreSQL 비밀번호로 수정하세요:

```env
DATABASE_URL=postgresql://postgres:실제비밀번호@localhost:5432/hos_contracts?schema=public
```

### 2. 데이터베이스 생성

다음 중 하나의 방법으로 데이터베이스를 생성하세요:

#### 방법 1: 배치 파일 사용
```cmd
데이터베이스생성.bat
```

#### 방법 2: pgAdmin 사용
1. pgAdmin 실행
2. PostgreSQL 서버 연결
3. 데이터베이스 우클릭 → Create → Database
4. 이름: `hos_contracts` 입력
5. Save

#### 방법 3: Prisma Migrate로 자동 생성
데이터베이스가 없어도 Prisma Migrate가 자동으로 생성하려고 시도합니다.
하지만 먼저 PostgreSQL 연결이 성공해야 합니다.

### 3. PostgreSQL 서비스 확인

Windows 서비스에서 PostgreSQL이 실행 중인지 확인:
1. `Win + R` → `services.msc`
2. `postgresql-x64-18` (또는 설치된 버전) 찾기
3. 상태가 "실행 중"인지 확인
4. 실행 중이 아니면 시작

### 4. 연결 테스트

다음 명령어로 PostgreSQL 연결을 테스트하세요:

```cmd
cd backend
데이터베이스연결확인.bat
```

## 🔧 빠른 해결

1. **`.env` 파일 확인**
   - `backend/.env` 파일이 있는지 확인
   - `DATABASE_URL`의 비밀번호가 실제 PostgreSQL 비밀번호와 일치하는지 확인

2. **데이터베이스 생성**
   ```cmd
   데이터베이스생성.bat
   ```

3. **다시 마이그레이션 실행**
   ```cmd
   npm run db:migrate
   ```

## 📝 참고

- PostgreSQL 기본 포트: `5432`
- 기본 사용자: `postgres`
- 데이터베이스 이름: `hos_contracts`
