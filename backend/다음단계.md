## 📋 다음 단계

### 1. Prisma 클라이언트 생성

```bash
npm run db:generate
```

이 명령어는 `prisma/schema.prisma` 파일을 기반으로 Prisma Client를 생성합니다.

### 2. 데이터베이스 마이그레이션

**먼저 PostgreSQL 데이터베이스가 실행 중인지 확인하세요!**

```bash
# 데이터베이스 생성 (PostgreSQL 접속 후)
psql -U postgres
CREATE DATABASE hos_contracts;
\q
```

**그 다음 마이그레이션 실행:**

```bash
npm run db:migrate
```

마이그레이션 이름을 입력하라고 나오면: `init` 또는 `initial` 입력

### 3. 서버 실행 테스트

```bash
npm run dev
```

정상적으로 실행되면:
- 콘솔에 "🚀 Server is running on port 3000" 메시지 표시
- http://localhost:3000/health 접속하여 확인

### 4. API 테스트

서버가 실행되면 다음 엔드포인트를 테스트할 수 있습니다:

**Health Check:**
```bash
curl http://localhost:3000/health
```

또는 브라우저에서: http://localhost:3000/health

## 🔍 문제 해결

### Prisma 클라이언트 생성 오류
```
Error: Cannot find module '@prisma/client'
```
**해결**: `npm install` 다시 실행

### 데이터베이스 연결 오류
```
❌ PostgreSQL connection error: ...
```
**해결**: 
1. PostgreSQL이 실행 중인지 확인
2. DATABASE_URL이 올바른지 확인 (.env 파일)
3. 데이터베이스가 생성되었는지 확인

### 포트 사용 중 오류
```
Error: listen EADDRINUSE: address already in use :::3000
```
**해결**: 다른 포트 사용하거나 기존 프로세스 종료
