# PostgreSQL 인증 오류 해결 가이드

## 🔴 현재 오류

```
Error: P1000: Authentication failed against database server
```

## ✅ 해결 단계

### 1단계: 환경 변수 확인

`backend` 폴더에서 다음 명령어 실행:

```cmd
환경변수확인.bat
```

이 스크립트가 다음을 확인합니다:
- `.env` 파일 존재 여부
- `DATABASE_URL` 설정
- PostgreSQL 연결 테스트
- 데이터베이스 존재 여부

### 2단계: .env 파일 생성 (없는 경우)

`.env` 파일이 없으면 다음 명령어 실행:

```cmd
환경변수생성_간단.bat
```

또는 수동으로 `backend/.env` 파일을 만들고 다음 내용 입력:

```env
DATABASE_URL=postgresql://postgres:실제비밀번호@localhost:5432/hos_contracts?schema=public
JWT_SECRET=faba88eb52cc638089b1e2529ada97111d2a85e2ac18f2579994ae161e602f36
```

**중요**: `실제비밀번호`를 PostgreSQL 설치 시 설정한 비밀번호로 변경하세요!

### 3단계: PostgreSQL 비밀번호 확인

`.env` 파일의 `DATABASE_URL`에서 비밀번호가 실제 PostgreSQL 비밀번호와 일치하는지 확인:

```env
DATABASE_URL=postgresql://postgres:password123@localhost:5432/hos_contracts?schema=public
                                 ^^^^^^^^^^^^
                                 이 부분이 실제 비밀번호와 일치해야 함
```

### 4단계: 데이터베이스 생성

데이터베이스가 없으면 생성:

```cmd
데이터베이스생성.bat
```

### 5단계: 다시 마이그레이션 실행

```cmd
npm run db:migrate
```

## 🔍 추가 확인 사항

### PostgreSQL 서비스 확인

1. `Win + R` 키 누르기
2. `services.msc` 입력 후 Enter
3. `postgresql-x64-18` (또는 설치된 버전) 찾기
4. 상태가 "실행 중"인지 확인
5. 실행 중이 아니면 시작

### 비밀번호를 모르는 경우

PostgreSQL 비밀번호를 잊어버린 경우:

1. **pgAdmin 사용**
   - pgAdmin 실행
   - 서버 연결 시 비밀번호 확인

2. **비밀번호 재설정**
   - `pg_hba.conf` 파일 수정 (고급)
   - 또는 PostgreSQL 재설치

## 📝 빠른 체크리스트

- [ ] `.env` 파일이 `backend/` 폴더에 있는가?
- [ ] `DATABASE_URL`의 비밀번호가 실제 PostgreSQL 비밀번호와 일치하는가?
- [ ] PostgreSQL 서비스가 실행 중인가?
- [ ] `hos_contracts` 데이터베이스가 존재하는가?

## 💡 팁

가장 흔한 원인은 **비밀번호 불일치**입니다. `.env` 파일의 `DATABASE_URL`을 다시 확인하세요!
