# 설치 및 실행 가이드

## ⚠️ Node.js 24 호환성 문제 해결

`@fastify/jwt`의 의존성인 `fast-jwt`가 Node.js 24를 지원하지 않아 `jsonwebtoken`으로 대체했습니다.

## 📦 설치

```bash
cd backend
npm install
```

## 🔧 환경 변수 설정

`.env` 파일을 생성하고 다음 내용을 입력:

```env
# Server
PORT=3000
NODE_ENV=development

# Database (Prisma)
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/hos_contracts?schema=public

# JWT
JWT_SECRET=your_jwt_secret_key_change_in_production
JWT_EXPIRES_IN=24h
JWT_REFRESH_SECRET=your_refresh_token_secret_key
JWT_REFRESH_EXPIRES_IN=7d

# AWS SES
AWS_REGION=ap-northeast-2
AWS_ACCESS_KEY_ID=your_aws_access_key
AWS_SECRET_ACCESS_KEY=your_aws_secret_key
EMAIL_FROM=noreply@yourdomain.com

# Frontend
FRONTEND_URL=http://localhost:3001

# Encryption
ENCRYPTION_KEY=your_encryption_key_change_in_production
```

## 🗄️ 데이터베이스 설정

### 1. PostgreSQL 데이터베이스 생성

```bash
# PostgreSQL 접속
psql -U postgres

# 데이터베이스 생성
CREATE DATABASE hos_contracts;

# 종료
\q
```

### 2. Prisma 마이그레이션

```bash
# Prisma 클라이언트 생성
npm run db:generate

# 마이그레이션 생성 및 적용
npm run db:migrate

# 마이그레이션 이름을 입력하세요 (예: init)
```

### 3. Prisma Studio (선택사항)

```bash
# 데이터베이스 GUI 실행
npm run db:studio
```

## 🚀 서버 실행

```bash
# 개발 모드 (자동 재시작)
npm run dev

# 프로덕션 모드
npm start
```

서버가 정상적으로 실행되면:
- http://localhost:3000/health 에서 확인 가능
- 콘솔에 "🚀 Server is running on port 3000" 메시지 표시

## 🔍 문제 해결

### npm audit 경고

일부 deprecated 패키지 경고는 무시해도 됩니다. 보안 취약점이 있다면:

```bash
# 자동 수정 (주의: breaking changes 가능)
npm audit fix

# 강제 수정 (권장하지 않음)
npm audit fix --force
```

### Node.js 버전 경고

`.npmrc` 파일에 `engine-strict=false`를 설정하여 경고를 무시할 수 있습니다. (이미 설정됨)
