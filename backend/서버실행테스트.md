# 서버 실행 테스트 가이드

## 1. 환경 변수 확인

`.env` 파일이 있는지 확인하고, 최소한 다음 변수들이 설정되어 있는지 확인:

```env
DATABASE_URL=postgresql://postgres:your_password@localhost:5432/hos_contracts?schema=public
JWT_SECRET=test_secret_key
JWT_REFRESH_SECRET=test_refresh_secret
FRONTEND_URL=http://localhost:3001
```

## 2. 서버 실행

```bash
npm run dev
```

## 3. 정상 실행 확인

서버가 정상적으로 실행되면:
- 콘솔에 "🚀 Server is running on port 3000" 메시지 표시
- http://localhost:3000/health 접속 시 JSON 응답 확인

## 4. 예상되는 오류

### 데이터베이스 연결 오류
```
❌ PostgreSQL connection error: ...
```
**해결**: PostgreSQL이 실행 중인지, DATABASE_URL이 올바른지 확인

### Prisma 클라이언트 오류
```
Error: Cannot find module '@prisma/client'
```
**해결**: 
```bash
npm run db:generate
```

### 포트 사용 중 오류
```
Error: listen EADDRINUSE: address already in use :::3000
```
**해결**: 다른 포트 사용하거나 기존 프로세스 종료

## 5. API 테스트

서버가 실행되면 다음 엔드포인트를 테스트할 수 있습니다:

### Health Check
```bash
curl http://localhost:3000/health
```

예상 응답:
```json
{
  "status": "ok",
  "message": "HOS Contract Management API is running"
}
```
